# Axure 原型 SVG 生成技能 (axure-prototype-svg)

## 技能简介
这是一个专为产品经理和交互设计师打造的 AI 技能。它能够根据你的需求文档或简单的文字描述，自动生成**高保真**且**完全兼容 Axure RP** 的 SVG 原型图。生成的 SVG 可以直接导入 Axure RP，并使用「Convert to Shapes（转换为形状）」功能完美转化为可编辑的原生形状，不会出现格式错乱或不支持的情况。

## 使用场景
- 根据需求文档快速生成产品原型草图，方便手动调整。
- 将需求文档中的交互规则自动映射到原型图中，提升效率，避免遗漏。

## 核心特性
- **Axure 完美兼容**：严格遵循 Axure SVG 解析的底层规则（如禁用 `<tspan>`、强制使用 6 位纯色 Hex 颜色码、使用坐标定位文本等），保证 100% 成功转换为形状。
- **规范的交互说明**：生成的画布不仅包含 UI 界面，还会自动在右侧扩展出「交互规则说明面板」，将业务逻辑、状态枚举、表单规则等清晰列出，方便研发和测试阅读。
- **严谨的绘制逻辑**：不凭空捏造业务字段，所有视觉元素均可回溯到需求点，缺失信息使用最小占位，确保原型严谨可靠。
- **多种风格可选**：默认采用中性、清晰、低装饰的企业后台风格，同时支持根据你提供的样式参考进行视觉定制（如替换主题色、圆角等）。

## 如何使用
1. **安装技能**：将本项目下载或复制到本地目录。使用支持 AI 辅助的编辑器（如 Trae、Cursor、Openclaw、Opencode 等）打开该目录，在 AI 对话框中让它帮你完成安装。例如输入：“请帮我安装 axure-prototype-svg 技能”。
2. **唤醒技能**：安装成功后，你可以通过在对话中输入以下任意**触发词**来随时唤醒它：`画原型`、`生成SVG原型`、`生成原型`、`绘制原型`等。

### 标准交互流程（分步进行）
为了保证生成的原型符合你的真实预期，本技能采用了**分步确认**的交互模式：
1. **第一步：信息收集**。唤醒技能后，AI 会主动询问并要求你提供必要的绘制信息（如页面名称、模块清单、表单项、交互规则等）。你需要补充这些需求细节。
2. **第二步：确认绘制计划**。AI 会根据收集到的信息，输出一份结构化的【绘制计划】（包含页面骨架、核心组件、交互状态、样式规范）。**此时你需要明确回复确认**，AI 才会进入下一步。
3. **第三步：正式绘制**。在你确认计划后，AI 将严格遵循 Axure 兼容规则生成 SVG 代码（左侧为 UI 界面，右侧为交互规则面板）。
4. **第四步：交付**。AI 会将经过自检的最终 SVG 代码交付给你。

## 提示词示例
你可以直接使用这样的提示词开始工作：
> "帮我**画原型**：需要一个后台的「员工管理」列表页。包含顶部搜索区（姓名输入框、部门下拉框、状态下拉框、查询/重置按钮），中间是数据表格（显示头像、姓名、工号、部门、入职时间、状态，操作列包含编辑和禁用），底部是分页器。请给出绘制计划。"

> "**生成SVG原型**：我需要一个移动端的登录页面。包含手机号输入框、验证码输入框及获取验证码按钮、登录按钮。请注意手机号需要校验11位数字格式，登录按钮在输入框为空时处于禁用状态（置灰）。"

## 在 Axure 中的使用指南
1. 将 AI 回复的 SVG 代码复制（复制<svg>……</svg>部分），并粘贴到Axure画布中。
2. 右键点击导入的图形，选择 **"Convert to Shapes"（转换为形状）**（或在顶部菜单中选择对应的转换选项）。
3. 转换完成后，该 SVG 就会变成 Axure 原生的矩形、文本和线条，你可以自由地修改文案、调整颜色、添加交互动作！
