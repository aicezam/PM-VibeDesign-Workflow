# 通用风格 Axure SVG 生成技能
本技能用于按需求文档生成可被 Axure RP「Convert to Shapes」稳定转换的高保真 SVG。
核心目标：风格可配置、结构不跑偏、交互可追溯、细节可复核、绝不乱画。

## Workflow 原型绘制流程（严格执行：必须分步与用户交互）

**第一步：信息收集（询问用户）**
- 开始绘制前，必须先主动询问用户提供必要的绘制信息（参考上文“输入要求”）。
- 若用户首轮提供的信息不完整，需提问引导用户补充核心需求（如页面包含哪些关键模块、是否有特定的交互状态要求等）。
- 阅读用户的需求后，主动询问并列出你还需要补充的信息。
- 🛑 **[强制停顿]**：在此步骤输出完毕后，必须结束当前回答，等待用户回复。

**第二步：确认绘制计划（等待用户确认）**
- 根据收集的信息，输出结构化的【绘制计划】（包含：页面骨架、核心组件、交互状态、样式规范）。
- 在结尾明确提问：“请确认是否按此计划开始绘制？（回复确认后我将开始写代码）”
- 🛑 **[强制停顿]**：必须等待用户明确回复确认后，才能进入第三步！严禁擅自开始生成 SVG！

**第三步：正式绘制（执行生成）**
用户确认计划后，按既定骨架与样式规范，严格遵循 Axure 兼容硬规则生成 SVG：
- 仅绘制计划中已映射的页面、模块、组件及交互状态；缺失信息用最小占位，不扩展业务含义。
- 默认采用通用企业后台风格（中性、清晰、低装饰）；若提供品牌模板，仅替换色板、字体、圆角等视觉层，保持结构与交互不变。
- 所有元素须对齐、间距、字号、状态色、禁用/hover/active 呈现跨模块一致；同类控件尺寸、语义状态颜色统一。
- **注意**：画布必须足够大（例如宽度 1440 或 1920），左侧画页面 UI，右侧必须画出对应的【交互规则说明】面板。
- 禁止新增需求外字段、按钮、流程、统计口径或接口数据；每处视觉元素均可回溯到需求点。
- 禁止猜测业务逻辑、流程，遇到问题可停止绘制，并向用户询问。

**第四步：自检与交付**
- 代码生成后，依据“输出前自检”清单进行代码和视觉核对。
- 确认无误后，输出最终的 SVG（页面视觉+交互规则） 代码交付给用户。

## 执行原则（先需求，后绘制）
1) 需求映射优先
  - 先抽取需求中的页面、模块、组件、状态、动作，再进入绘制。
  - 每个视觉元素都必须能映射到需求点；无法映射则不绘制。
2) 默认通用风格
  - 未指定品牌时，采用通用企业后台风格（中性、清晰、低装饰）。
  - 若用户给出单独风格模板，仅替换视觉样式规范，不改变信息结构与交互逻辑。
3) 细节一致性
  - 对齐、间距、字号、行高、状态色、禁用态、hover/active 呈现保持一致。
  - 同类控件尺寸统一，不同模块的相同语义状态颜色统一。 
4) 禁止乱画
  - 不凭空添加业务字段、按钮、流程、统计口径、接口数据。
  - 缺失信息仅使用最小占位，不扩展业务含义。

## 输入要求（尽量收集）
- 页面名称与页面类型（列表、表单、详情、抽屉、弹窗等）
- 需求文档中的模块清单与优先级
- 表单项/表格列/操作按钮/状态枚举
- 交互规则（点击、切换、展开、关闭、禁用、校验、空态、异常态）
- 文案与数据来源限制（可占位字段、不可编造字段）
- 是否有风格模板（若有，作为视觉参考而非结构约束）

## Axure 兼容硬规则（必须）
1. 文本定位
  - 所有文本使用 `<g transform="translate(x,y)">` 包裹。
  - `<text>` 不写绝对 `x/y`，或只用 `x="0" y="0"`。
2. 文本结构
  - 禁止 `<tspan>`。
  - 多行文本使用多段独立 `<g><text>...</text></g>`。
3. 样式方式
  - 禁止 CSS 类、`<style>`、外链字体。
  - 全部样式使用内联属性。
4. 对齐属性
  - 禁止 `text-anchor`、`dominant-baseline`。
  - 居中效果通过坐标计算实现。
5. 图形能力边界
  - 禁止 `mask`、`clipPath`、`filter`、`foreignObject`、`use`。
  - 优先使用 `rect/line/path(简化)`，避免复杂特效。
6. 颜色与透明度格式
  - 绝对禁止使用 8 位 Hex 颜色码（例如 `#000000A6` 或 `#1F232980`），Axure 无法解析，会导致转形状失败或颜色异常。所有颜色必须且只能使用标准的 6 位 Hex 颜色码（例如 `#000000`、`#FFFFFF`、`#1677FF`）。
  - 绝对禁止使用 `rgba()` 格式。
  - 如果需要透明度效果 （如蒙层、禁用态、半透明文字），请直接通过与底色混合计算，换算出一个个个纯色的 6 位 Hex 近似颜色个个。例如：在白底上的 65% 黑，请直接使用 `#A6A6A6`，而不要写 `#000000` 加透明度。

## 原型交互规则说明编写规范（纯文本，右侧面板）
**【呈现方式要求】**
- 必须在原型页面区域之外，向右扩展 SVG 画布宽度，在右侧独立的空白区域使用 `<rect>` 绘制一个背景板作为“交互规则面板”。
- 交互规则面板绝对不能与原型的 UI 界面重叠，不能挤占原型空间，确保不影响原型本来的内容和布局。
- 将下述业务逻辑、状态枚举、表单规则等，精炼提炼后，使用多行 `<g><text>...</text></g>` 绘制在该面板中（可参考底部的极简示例）。

1. 业务逻辑与数据规则
提取需求中的核心规则（数据来源、目标、对象、动作、计算公式、重置逻辑等）；禁止编造，缺项用“待补充”占位。
2. 交互状态映射
按组件列举：触发动作（如点击/滑动） → 反馈（如缩放/高亮） → 状态/结果；确保操作可逆（如展开必可关闭），组件联动清晰。
3. 异常与边界状态
列出全局或局部异常态表现：弱网/无网络、数据加载中、空数据、权限不足、错误边界等。
4. 表单规则（有表单时）
逐字段列出：名称、必填项、类型、长度/格式限制、默认值、可编辑条件；校验时机及错误文案位置；字段间依赖（显隐、清空、禁用、互斥）。
5. 提交按钮规则（有表单提交按钮时）
说明按钮状态（可点、不可点、提交中、成功、失败）；可点条件（必填+格式+业务前置全通过）；点击反馈（loading+防重；成功→跳转/刷新/关闭，失败→留输入并提示）；回车提交、节流等策略按需求说明。
6. 列表状态-操作枚举（有列表且存在状态和操作时）
列出列表表格中，不同状态的数据的操作枚举以及交互交互状态映射、操作说明，例如：
- 正常状态：编辑、删除
- 禁用状态：无操作
- 异常状态：重试、联系客服
7. 筛选规则（有筛选功能时）
说明筛选条件（字段、操作符、值）；筛选触发时机（如点击筛选按钮、输入完成后自动筛选）；筛选结果展示（如筛选后的数据表格、筛选条件的可视化表示）；清空筛选（如点击清空按钮、切换筛选条件后自动清空）。
8. 分页规则（有分页功能时）
说明分页方式（如数字分页、上一页下一页）；每页显示条数；跳转至指定页的输入框；跳转按钮位置（如在分页区域或结果区域）。
9. 排序规则（有排序功能时、列表有不同状态数据时）
说明排序字段（如名称、时间、数量等）；排序方向（如升序、降序）；排序触发时机（如点击表头、切换排序字段后自动排序）；排序结果展示（如表格数据按排序字段排序）。
10. 空态规则
说明空数据的表现（如显示“暂无数据”、“暂无相关结果”等）；空数据的触发时机（如列表为空、搜索无结果等）；空数据的处理方式（如提供添加数据的链接、提示用户联系客服等）。
11. 其他规则
说明其他交互规则（如滚动加载、下拉刷新、长按操作等）；特殊场景下的注意事项（如数据加载延迟、权限限制等）。

## 通用视觉样式规范（默认模板）
**字体：**
- `font-family="Arial, sans-serif"`
- 标题 16，正文 14，辅助 12

**颜色：**
- 主色 `#1677FF`
- 页面背景 `#F5F7FA`
- 卡片背景 `#FFFFFF`
- 主文字 `#1F2329`
- 次文字 `#4E5969`
- 占位 `#C9CDD4`
- 边框 `#E5E6EB`
- 分割线 `#F2F3F5`

**状态色：**
- 成功：字 `#00B42A`，底 `#E8FFEA`，边 `#7BE188`
- 进行中：字 `#1677FF`，底 `#E8F3FF`，边 `#91C9FF`
- 警告：字 `#FF7D00`，底 `#FFF3E8`，边 `#FFB65D`
- 失败：字 `#F53F3F`，底 `#FFECE8`，边 `#FF9A8B`

**尺寸与间距：**
- 基础网格 8
- 页面内边距 24，卡片内边距 16
- 输入/按钮高度 32，表格行高 40
- 常规圆角 4，小标签圆角 2

**页面骨架（通用后台）：**
- 桌面端画布建议 `1440 x 900`（可扩展高度）
- 手机端画布建议：`390 x 844`、`375 x 812`、`360 x 800`
- 顶部栏（全局信息/操作）
- 次级导航（Tabs 或面包屑，按需求二选一）
- 内容区（卡片化）：筛选区 + 结果区 + 分页
- 可选层：弹窗/抽屉/提示条/空态块

**组件规范（默认值）：**
- 输入框/下拉：`height=32`，白底，1px 边框
- 主按钮：`height=32`，主色底白字
- 次按钮：`height=32`，白底灰边深字
- 状态标签：高 22~24，左右内边距 8
- 进度条：轨道高 6，进度同高同圆角
- 分页：按钮 `32x32`，选中态主色反白

## 单独风格模板参考（可替换样式规范）
当用户提供单独风格模板时，只替换以下层：
- 色板、字体、圆角、描边粗细、阴影表现

保持不变：
- 页面结构层级
- 交互流程与状态流转
- Axure 兼容硬规则

## 输出前自检（防乱画）
- [ ] 每个元素可回溯到需求点，无无源组件
- [ ] 所有文本均符合 `<g><text>` 规则，无 `<tspan>`
- [ ] 无 CSS 类、无 `<style>`、无禁用特性标签
- [ ] 颜色代码全部是 6 位 Hex（如 `#FFFFFF`），没有 8 位 Hex（如 `#000000A6`），没有 `rgba()`  
- [ ] 对齐、间距、字号、状态色跨模块一致
- [ ] 交互状态齐全且闭环，不存在悬空流程
- [ ] 缺失信息仅用占位，不扩写业务逻辑
- [ ] 是否严格遵循了分步确认原则？（如果没有等待用户确认就走到了这一步，立刻停止并向用户道歉）。
- [ ] SVG 画布中是否已包含纯文本的【交互规则说明】面板？且覆盖了状态、表单、按钮等核心规则？
- [ ] 检查是否缺少页面，如果缺少，立即停止并询问用户是否绘制

## 输出与命名
- 每个页面一个独立 SVG 文件，每个模块一个独立文件目录
- 文件名与页面名称一致
- 目录保持模块化，便于按功能导入 Axure
