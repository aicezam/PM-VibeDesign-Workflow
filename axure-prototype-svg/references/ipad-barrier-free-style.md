# iPad 端适老化与无障碍(Barrrier-Free)视觉样式规范

本规范作为 `axure-prototype-svg` 技能的**单独风格模板**，专为运行在 iPad (或类似大屏平板设备) 上的医患交互系统设计。
核心设计理念：**字大、高对比度、防误触、操作极简、步骤清晰**，确保患者（尤其是中老年患者）能轻松看清内容并完成交互确认。

## 1. 视觉样式规范 (Visual Styles)

### 1.1 字体 (Typography)
- **字体家族**: `font-family="Arial, sans-serif"`
- **字号 (显著放大)**:
  - **核心大标题 (如“手术知情同意书”)**: 32px (加粗)
  - **模块标题/重点提示**: 28px (加粗)
  - **正文/列表内容**: 24px (标准)
  - **辅助说明/按钮文案**: 20px
- **行高**: 默认 `1.8`，保证段落文字阅读不拥挤。
- **字重**: 关键信息必须使用 `font-weight="bold"` 进行强调。

### 1.2 色彩 (Colors)
采用高对比度色彩体系，避免使用难以辨认的渐变或低对比度配色。

- **品牌/主操作色 (Primary)**: `#1677FF` (高亮蓝，用于主按钮、进度条)
- **中性色 (Neutral)**:
  - 页面背景 (Background): `#F5F7FA` (浅灰底色，防刺眼)
  - 核心内容区背景 (Surface): `#FFFFFF` (纯白卡片，突出内容)
  - 主文本色 (Text Main): `#1F2329` (深黑，极高对比度)
  - 次文本色 (Text Secondary): `#4E5969` (深灰)
  - 边框/分割线 (Border): `#E5E6EB`
- **功能色 (Semantic)**:
  - 成功/已完成 (Success): `#00B42A` (用于打钩图标、已完成状态)
  - 警告/待确认 (Warning): `#FF7D00` (用于需要患者注意的地方)
  - 危险/错误 (Danger): `#F53F3F`

### 1.3 尺寸与间距 (Sizes & Spacing)
- **屏幕基准**: `1024 x 768` (iPad 横屏模式为主) 或 `768 x 1024` (竖屏模式)。本场景推荐**横屏**展示以提供更宽广的视野。
- **点击区域 (Touch Target)**: 
  - **极简大触控区**：任何可点击的按钮或卡片，高度不低于 `64px`，宽度不低于 `120px`。
  - **勾选框/单选框**：尺寸至少 `40px x 40px`。
- **圆角 (Border Radius)**:
  - 卡片/弹窗圆角: 16px (显得更加亲和安全)
  - 按钮圆角: 8px 或 32px (胶囊型大按钮)
- **间距 (Spacing)**:
  - 页面左右外边距: 40px
  - 卡片内边距: 32px
  - 模块间距: 24px (确保内容不堆叠)

## 2. 组件风格与使用场景 (Components)

### 2.1 导航与布局 (Navigation & Layout)
- **顶部状态栏 (Header)**: 高度 `80px`。左侧放返回或取消按钮（带大图标+文字），中间放置明确的大字号当前步骤标题（如“第一步：病情介绍”）。
- **底部操作栏 (Footer)**: 固定在屏幕底部，高度 `100px`，白底带顶部阴影。放置全局的“下一步”、“确认并签字”等核心大按钮。
- **分步进度条 (Steps)**: 推荐使用大尺寸的横向步骤条，当前步骤高亮显示，已完成步骤显示大号的 `✓`，未开始步骤置灰。

### 2.2 数据录入与操作 (Data Entry)
- **主操作按钮 (Primary Button)**:
  - 尺寸：高度 `64px`，宽度可占满半屏或全屏。
  - 样式：蓝底白字，字号 `24px`，无边框，带微小阴影以凸显可点击性。
- **次要按钮 (Secondary Button)**:
  - 样式：白底黑字，浅色粗边框 (`2px`)，字号 `24px`。
- **大号勾选框 (Checkbox)**:
  - 尺寸：`40px x 40px` 的方框，选中后内部出现粗线条的 `✓`。
  - 交互：点击整行（包含文字和勾选框）都可以触发勾选，防误触。

### 2.3 数据与内容展示 (Data Display)
- **内容卡片 (Card)**: 
  - 纯白背景，圆角 `16px`，带有 `1px` 的 `#E5E6EB` 边框和非常柔和的阴影。
  - 卡片内部文字必须留有充足的四周留白（Padding `32px`）。
- **视频播放区 (Video Player)**:
  - 在“播放视频”环节，视频播放器应占据屏幕核心位置（如 `800 x 450`）。
  - 播放控制按钮（播放、暂停、全屏）需显著放大，至少 `64px x 64px`。
- **双路录制画面提示 (Recording Status)**:
  - 在屏幕角落（如右上角）保留一个悬浮的录制状态指示器（如“🔴 正在录音录像 02:15”），背景半透明，字号 `20px`，既能起提示作用又不遮挡核心阅读内容。

### 2.4 反馈与确认 (Feedback)
- **居中弹窗 (Dialog)**: 
  - 宽度占屏幕的 `60%` 左右，带半透明黑色蒙层。
  - 用于关键节点的二次确认（如“您确认已清楚了解手术风险吗？”）。
  - 弹窗内的“确认”和“取消”按钮必须是等大的 `64px` 高度按钮。
- **签字板 (Signature Pad)**:
  - 若有签字环节，提供一块巨大的空白区域（至少 `600 x 300`），背景色为极浅的灰色（`#FAFAFA`）以区分可绘制区域。
  - 旁边配备显眼的“重签”和“确认”按钮。
- **语音播报辅助 (Voice Assistant - 视觉示意)**:
  - 针对视力不佳的患者，界面上可配备一个显眼的“喇叭”图标（大小 `48px`），旁边配有“朗读本页”文字，点击后按钮呈现播放波纹动效。
